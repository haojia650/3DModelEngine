TopoDS_Shape 几何实体数据，用于描述模型零件本身；2. AIS_Shape 显示对象数据，用于将几何对象加载到场景中并支持选择与显示；3. AIS_InteractiveContext 交互上下文数据，用于完成对象选择、刷新和显示控制；4. V3d_View 视图数据，用于完成三维场景浏览与相机控制；5. gp_Trsf、gp_Vec 等几何变换数据，用于对象的平移、旋转和缩放计算。本次实验我没有采用 Rhino 插件形式，而是直接采用 OCC 库开发独立桌面程序。这样虽然比插件开发更费力，但优点是能直接控制底层几何对象和交互逻辑，更适合本次实验“核心功能算法设计”的要求，也与我前面已经完成的程序内容保持一致。从产品设想上看，这个小软件可以扩展为一个轻量级三维展示与交互原型平台，用于课程演示、小型结构展示、教学实验或后续 CAD/CAE 功能验证，具有一定的教学和工程原型价值。实验过程实验环境与工程基础1. 开发环境操作系统：Windows 11开发工具：Visual Studio 2022图形库：OpenCASCADE Technology（OCCT）工程框架：MFC 文档/视图结构编程语言：C++2. 工程基础说明本次实验直接基于前面已经完成的 OCCT 桌面程序工程进行扩展。该工程已经具备 Viewer、AIS_InteractiveContext 和三维模型显示基础，因此本次实验重点不再是“从零搭建窗口”，而是围绕核心交互功能进行算法设计、代码实现与总结分析。软件总体设计本软件采用 MFC 文档/视图结构，并结合 OCCT 进行三维场景管理。1. COccResurfDoc 负责创建三维查看器、维护交互上下文、创建模型对象；2. COccResurfView 负责处理鼠标、键盘和场景刷新；3. AIS_Shape 作为场景中被选中和被变换的对象；4. TransformState 用于保存每个对象的平移、旋转和缩放状态。该结构的优点是职责划分清晰：文档类负责“数据与模型”，视图类负责“交互与显示”，有利于后续继续扩展更多功能。核心功能实现1. 文档类头文件设计（OccResurfDoc.h）代码选取：Handle(AIS_InteractiveContext) myAISContext;Handle(V3d_Viewer) myViewer;Handle(V3d_Viewer) GetViewer(void) const { return myViewer; }Handle(AIS_InteractiveContext) GetAISContext(void) const { return myAISContext; }const std::vector<Handle(AIS_Shape)>& GetModelShapes() const { return myModelShapes; }void UpdateModelCenter();void DrawSphere(double Radius);代码分析：该部分代码定义了程序文档类中的核心对象。myViewer 负责三维查看器管理，myAISContext 负责对象选择和交互控制，myModelShapes 用于维护场景中的所有几何零件，UpdateModelCenter() 和 DrawSphere() 则分别用于更新模型中心和完成模型创建。通过这些声明，程序建立了“场景数据层”的基础。2. 文档类实现（OccResurfDoc.cpp）代码选取：myViewer = new V3d_Viewer(theGraphicDriver);myViewer->SetDefaultLights();myViewer->SetLightOn();myAISContext = new AIS_InteractiveContext(myViewer);myAISContext->SetDisplayMode(AIS_Shaded, true);myAISContext->SetAutomaticHilight(Standard_True);代码分析：这部分代码完成 3D 查看器和交互上下文的初始化。Viewer 用于承载整个三维显示环境，AIS_InteractiveContext 负责后续的对象选择和刷新控制。设置默认灯光和着色模式后，模型可以以较正常的三维效果显示出来，这是整个程序能够运行的基础。3. 模型构建算法代码选取：auto DrawShape = [&](const TopoDS_Shape& theShape,    const gp_Trsf& theTrsf,    const Quantity_Color& theColor)    {        TopoDS_Shape aTransShape = BRepBuilderAPI_Transform(theShape, theTrsf, Standard_True);        Handle(AIS_Shape) anAis = new AIS_Shape(aTransShape);        anAis->SetColor(theColor);        myModelShapes.push_back(anAis);        myAISContext->Display(anAis, Standard_False);    };代码分析：这段代码通过一个局部辅助函数统一封装了“几何体创建 + 变换 + 上色 + 显示”的流程。它的作用是减少重复代码，并把每个几何零件都记录到 myModelShapes 中，方便后续进行选择和变换处理。该设计体现了代码模块化思想，也使模型构建过程更清晰。4. 视图初始化与显示代码选取：myView = GetDocument()->GetViewer()->CreateView();myView->SetShadingModel(V3d_GOURAUD);Handle(WNT_Window) aWntWindow = new WNT_Window(GetSafeHwnd());myView->SetWindow(aWntWindow);GetDocument()->DrawSphere(6.0);FitAll();代码分析：这部分代码用于在视图类中建立窗口与三维视图之间的联系。程序先创建 View，再将 MFC 窗口与其绑定，最后调用 DrawSphere() 把模型显示出来，并使用 FitAll() 自动调整视角。这样程序启动后即可直接看到完整三维场景。5. 单个对象选择算法代码选取：ctx->MoveTo(point.x, point.y, myView, Standard_False);ctx->SelectDetected(AIS_SelectionScheme_Replace);代码分析：这里实现的是鼠标点击选择逻辑。MoveTo() 先根据屏幕坐标检测当前鼠标下方的几何对象，SelectDetected() 再把检测到的对象设为当前选中对象。该过程实现了从二维点击位置到三维对象选择的映射，是整个交互系统最重要的入口之一。6. 变换状态保存算法代码选取：struct TransformState{    gp_Vec translation;    Standard_Real rotationX;    Standard_Real rotationY;    Standard_Real rotationZ;    Standard_Real uniformScale;};代码分析：程序并没有把每次拖拽直接永久写入几何模型，而是先通过 TransformState 保存对象当前的平移、旋转和缩放状态。这样做的好处是可以支持临时变换预览、重置变换和应用变换，便于后续继续扩展撤销和历史记录功能。7. 对象变换矩阵组合算法代码选取：const gp_Pnt aCenter = GetShapeCenter(theShape);gp_Trsf aScale;aScale.SetScale(aCenter, aState.uniformScale);gp_Trsf aRotateX;aRotateX.SetRotation(gp_Ax1(aCenter, gp::DX()), aState.rotationX);gp_Trsf aRotateY;aRotateY.SetRotation(gp_Ax1(aCenter, gp::DY()), aState.rotationY);gp_Trsf aRotateZ;aRotateZ.SetRotation(gp_Ax1(aCenter, gp::DZ()), aState.rotationZ);gp_Trsf aTranslate;aTranslate.SetTranslation(aState.translation);代码分析：该段代码体现了本次实验最核心的算法思想：围绕对象自身中心进行缩放和旋转，而不是围绕全局原点。先求出对象的中心点，再分别构造缩放矩阵、绕 X/Y/Z 轴旋转矩阵和平移矩阵，最后组合为最终变换结果。这保证了对象在交互时具有更自然的视觉效果。8. 移动拖拽方向优化算法代码选取：const Handle(Graphic3d_Camera)& aCamera = myView->Camera();const gp_Vec aRight(aCamera->SideRight());const gp_Vec aUp(aCamera->OrthogonalizedUp());moveVec = aRight.Multiplied(static_cast<Standard_Real>(-delta.x) * kMoveStepPerPixel)    + aUp.Multiplied(static_cast<Standard_Real>(-delta.y) * kMoveStepPerPixel);代码分析：这部分代码的作用是把屏幕上的鼠标拖拽方向映射到当前相机视角下的右方向和上方向，从而解决“三维场景中拖拽方向与视觉方向不一致”的问题。相比直接沿世界坐标轴平移，这种做法更符合用户直觉，交互体验更稳定。9. 变换结束自动退出模式算法代码选取：else if (wasTransforming){    ExitTransformMode();}代码分析：该代码用于优化交互流程。用户完成一次对象拖拽后，系统会自动退出当前变换模式，但保留对象选中状态。这样既能避免继续误拖对象，又能保证用户可以直接旋转视图观察整体效果，提高了操作流畅度。代码实现说明本次实验中的核心功能已经在前面的 OCCT
